# 🛍 Telegram Store SaaS — Multi-tenant Platform

Платформа для продажи Telegram мини-магазинов. Каждый клиент получает свой изолированный магазин, бота и панель управления.

---

## 📁 Структура проекта

```
telegram-saas/
├── server.js          — SaaS бэкенд (Express + JWT auth)
├── db.js              — JSON база данных с изоляцией по owner
├── bot-runner.js      — Запускает отдельный бот для каждого магазина
├── auth.html          — Регистрация / вход для владельцев магазинов
├── dashboard.html     — Панель управления магазином (для каждого клиента)
├── superadmin.html    — Ваша панель (все магазины, статистика)
├── store.html         — Mini App магазин (multi-tenant, ?owner=ID)
├── package.json
└── data/              — Создаётся автоматически
    ├── owners.json    — Все зарегистрированные владельцы
    └── stores/
        └── {ownerId}/
            ├── products.json
            ├── orders.json
            └── config.json
```

## ⚡ Быстрый старт

### 1. Установите зависимости
```bash
npm install
```

### 2. Настройте конфиг
В `server.js` найдите CONFIG и замените:
```js
JWT_SECRET:            'замените-на-длинную-случайную-строку',
SUPER_ADMIN_EMAIL:     'ваш@email.com',
SUPER_ADMIN_PASSWORD:  'ВашСильныйПароль123',
```

### 3. Запустите сервер
```bash
# Терминал 1 — основной сервер
node server.js

# Терминал 2 — бот-раннер (запускает ботов для каждого клиента)
node bot-runner.js
```

### 4. Откройте в браузере
```
http://localhost:3000/auth.html        — регистрация/вход клиентов
http://localhost:3000/superadmin.html  — ваша панель суперадмина
```

## 🌐 Как это работает

### Для каждого клиента (владельца магазина):

1. Клиент регистрируется на `auth.html`
2. Попадает в `dashboard.html` — там он управляет товарами и заказами
3. Вставляет свой Bot Token и Chat ID (из @BotFather и @userinfobot)
4. `bot-runner.js` автоматически запускает бота для него
5. Его магазин доступен по ссылке:
   ```
   https://ваш-сервер.com/store.html?owner=ИД_КЛИЕНТА
   ```

### Для вас (суперадмин):

- `superadmin.html` — видите все магазины, заказы, выручку
- Можете активировать/деактивировать любой магазин одной кнопкой
- Можете менять тариф (Trial → Active → Suspended)

## 🚀 Деплой на Render

### 1. Залейте на GitHub
```bash
git init && git add . && git commit -m "init"
git remote add origin https://github.com/ваш/telegram-saas.git
git push -u origin main
```

### 2. Создайте Web Service на render.com
- **Build:** `npm install`
- **Start:** `node server.js`

### 3. Добавьте переменные окружения
| Переменная | Значение |
|------------|----------|
| `JWT_SECRET` | длинная случайная строка |
| `SUPER_ADMIN_EMAIL` | ваш email |
| `SUPER_ADMIN_PASSWORD` | ваш пароль |
| `SERVER_URL` | https://ваш-сервер.render.com |

### 4. Для bot-runner — отдельный сервис
Создайте второй Web Service с тем же репозиторием:
- **Start:** `node bot-runner.js`
- Те же env переменные + `SERVER_URL`

## 💰 Как брать деньги с клиентов

На данный момент — вручную (рекомендуется на старте):

1. Клиент регистрируется → получает магазин
2. Вы в суперадмин видите нового клиента (тариф: Trial)
3. Клиент платит вам через Click/Payme/Telegram/наличные
4. Вы переключаете его тариф на **Active**
5. Если не платит → **Suspended** → магазин отключается

Позже можно подключить Stripe или Payme для автооплаты.

---

## 🔐 Безопасность

- Пароли хранятся через bcrypt (хэш, не текст)
- JWT токены для аутентификации (30 дней)
- Каждый владелец видит только свои данные
- Суперадмин отделён от обычных владельцев

---

## 📊 Тарифы (настройте сами)

| Тариф | Значение |
|-------|----------|
| `trial` | Пробный период — магазин активен |
| `active` | Оплачен — магазин активен |
| `suspended` | Не оплачен — магазин отключён |

Смена тарифа — одна кнопка в superadmin.html.
